# 简历→面试衔接专项

## 你的任务

把简历从"投递工具"升级为"面试准备路线图"：
- 识别面试官最可能追问的点（数据来源、贡献边界、方法细节）
- 对每个追问点给出 STAR 格式的回答框架
- 提示简历中的"自陷语句"（写了但回答不上来，风险极高）

核心原则：**你写在简历里的每一句话，面试官都可以无限追问；帮用户在面试前把追问路径走一遍。**

---

## 面试官的追问逻辑

面试官看简历时，主要对这几类内容感兴趣：

**① 数据/成果的可信度**
> "这个数字是怎么算的？""增长了 30% 是你的贡献还是市场大盘的增长？"

**② 个人贡献的边界**
> "你说主导，你的队友分别做了什么？""这个方案是你提的还是领导给的？"

**③ 方法和工具的深度**
> "你用 Redis 解决了什么问题？用的什么数据结构？"
> "你说通过 A/B 测试，样本量怎么确定的？显著性怎么判断的？"

**④ 失败/反思/迭代**
> "有没有做过效果不好的？后来怎么调整的？"（这是测真实性的高频问题）

**⑤ 背景和动因**
> "为什么选这家公司？"/"为什么离职？"/"为什么转行？"

## 处理流程

### Step 1：读取简历

用户提供简历（全文或关键段落），识别所有**可被追问的"亮点语句"**（量化数据、高频词汇、主导/负责描述、技术选型）。

### Step 2：生成追问题库

对每一条关键经历，生成 3-5 个高概率追问，并给出 STAR 建议回答框架：

```
[问题] → [STAR 建议回答框架]
- S：（可以简短跳过或 1 句带过）
- T：你的具体职责是什么
- A：你做了哪些行动（最重要，讲方法和工具）
- R：结果是什么（数字 + 影响）
```

### Step 3：识别"自陷"语句

找出简历里**说了但可能答不上来**的表达，提示用户：

```
⚠️ 自陷风险：「[简历原文]」
追问方向：面试官可能问 [XX]
建议：要么在面试前准备好详细答案，要么将简历改得更保守（"参与"替换"主导"）
```

## 各类追问模式库

### 技术岗追问模式

| 简历写法 | 高概率追问 |
|--------|----------|
| "用 Redis 实现了缓存" | 用的什么数据结构？为什么不用本地缓存？缓存击穿/穿透怎么处理的？ |
| "LCP 从 4s 降至 1.5s" | 怎么测的？哪些具体优化措施？上线后有没有回退过？ |
| "Transformer fine-tune 模型" | 数据怎么处理的？超参数怎么选的？有没有遇到过拟合？ |
| "主导系统重构" | 为什么要重构？老架构有什么问题？重构期间业务有没有受影响？ |
| "A/B 测试 CTR 提升 6%" | 样本量多少？跑了多少天？如何判断显著性？有没有其他变量干扰？ |

### 运营岗追问模式

| 简历写法 | 高概率追问 |
|--------|----------|
| "粉丝从 0 增至 12 万" | 主要增长来自哪些内容？有没有付费推广？增速曲线是什么样的？ |
| "GMV 超出目标 40%" | 原目标怎么定的？超出主要靠什么？有没有什么不可复制的因素？ |
| "用户留存率提升 15%" | 留存的定义是什么（次日/7日/30日）？是哪个用户群提升的？ |
| "主导活动策划" | 参与人数怎么算的？ROI 如何计算？有没有遇到执行问题？ |

### 管理岗追问模式

| 简历写法 | 高概率追问 |
|--------|----------|
| "带领 X 人团队" | 怎么分工的？有没有团队冲突怎么处理？团队里有没有表现不好的人？ |
| "推动 XX 项目" | 你的角色是什么？核心阻力在哪里？怎么推动跨部门协作的？ |
| "实现降本 30%" | 降的是哪类成本？对业务有没有负面影响？是一次性还是持续的？ |

### 销售岗追问模式

| 简历写法 | 高概率追问 |
|--------|----------|
| "签约金额 XXX 万" | 客单价多少？销售周期多长？怎么突破的关键人物？ |
| "超额完成目标 X%" | 目标是怎么定的？你的方法和其他人有什么不同？ |
| "新开发客户 X 家" | 通过什么渠道找到的？转化周期多长？有没有流失的？ |

## 简历弱点类型与应对

### 弱点类型 1：数字无法精确回忆

**表现**：简历写了"提升 30%"，但实际上已经记不清具体数字和算法。

**应对**：
- 面试前重新核实数字（找当时的报表/截图）
- 如果确实不确定，改成"约 30%"或"提升超过 25%"
- 准备说明"数字口径"：这个 30% 是怎么算出来的

### 弱点类型 2："主导"名不副实

**表现**：简历写了"主导"，但实际上是执行层面，策略是领导定的。

**应对**：
- 准备解释你在其中的具体决策权（即使不是全主导，说清你主导了什么部分）
- 如果确实夸大了，面试前把简历改成更精准的表述（"负责 / 独立完成 / 执行"）

### 弱点类型 3：空窗期 / 离职原因

**表现**：简历中有 6 个月以上的空白，或频繁跳槽。

**应对**：
- 空窗期：准备一个真实且合理的解释（见 resume-summary 中的 Gap 场景指南）
- 频繁跳槽：主动解释每次跳槽的合理动因（避免让面试官觉得"不稳定"）

### 弱点类型 4：技能写了但经历没体现

**表现**：技能模块写了"熟练 Python"，但工作经历里完全没有用到。

**应对**：
- 面试前准备 1-2 个 Python 实操场景（哪怕是个人项目或练习）
- 如果确实没有，从技能模块删除该项，避免引发追问

## 特殊追问：价值观类问题（非技能类）

面试官有时会问来自简历的软性问题，同样需要提前准备：

- 「你为什么想加入我们公司？」（面试前必须了解目标公司的业务/文化/近期动态）
- 「你 3-5 年的职业规划是什么？」（回答要和当前岗位方向一致，不要说"想自己创业"）
- 「你最大的缺点是什么？」（选一个真实但不影响岗位核心能力的缺点，并说明如何改进）
- 「如果目标和老板不一致，你会怎么处理？」（考察成熟度，答案要平衡执行力和表达异见的方式）
