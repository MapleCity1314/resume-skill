# 各职业领域简历指南

本文档提供各主流职业方向（含技术岗）的：关键技能词汇、模块侧重、量化指标参考、以及经历描述示例。

---

## 0. 计算机技术方向总览

技术类岗位简历的通用要点：
- **量化指标**以性能数据为主：QPS、延迟（P99）、包体积、准确率、A/B 效果
- **技术选型**要写出理由，体现思考（「选用 X 而不是 Y，因为...」）
- **项目链接**：GitHub / 在线访问地址能大幅提升可信度
- **规模背书**：写清楚日活/MAU/数据量/并发量，说明你在真实生产环境做过

## 0.1 前端开发

### 核心技能词汇
HTML5 / CSS3 / JavaScript（ES2022+）/ TypeScript / React（Hooks/Fiber/并发特性）/ Vue 3（Composition API）/ Vite / Webpack / 微前端（qiankun/Module Federation）/ Web Performance（LCP/FCP/CLS）/ PWA / SSR/SSG（Next.js/Nuxt）/ WebSocket / Canvas/WebGL / 单元测试（Jest/Vitest）/ E2E 测试（Playwright/Cypress）

### 量化指标参考
- 首屏加载时间（LCP ms）
- Lighthouse 评分提升（X → Y）
- 包体积减少量（MB 或 %）
- 页面跳出率降低（%）
- 组件库复用覆盖的项目数
- 负责产品的 DAU/MAU

### 描述示例
- **弱**：「参与公司官网的前端开发。」
- **强**：「主导官网重构（React 18 + Vite），引入 Route-level Code Splitting 与图片懒加载，
  LCP 从 4.1s 降至 1.3s，Lighthouse 性能评分从 52 提升至 93，
  移动端跳出率下降 28%，月 UV 提升 15%。」

## 0.2 后端开发

### 核心技能词汇
Java（JUC/JVM 调优/设计模式）/ Go（goroutine/channel/GMP）/ Python（asyncio/FastAPI）/ Spring Boot / Spring Cloud / Gin / gRPC / MySQL（索引优化/分库分表/事务隔离级别）/ Redis（数据结构/持久化/集群/缓存穿透击穿）/ Kafka（幂等消费/分区策略/延迟队列）/ Elasticsearch / Docker / K8s（基础）/ 分布式事务（Seata/TCC）

### 量化指标参考
- QPS / TPS（提升前后对比）
- 接口响应时间（P99 ms）
- 系统可用性（SLA %）
- 数据规模（日订单量/数据条数）
- 存储成本节约量
- 服务数量/微服务拆分规模

### 描述示例
- **弱**：「负责优化了系统性能。」
- **强**：「负责订单服务性能优化，通过引入 Redis 缓存 + 异步消息队列（Kafka），
  在日订单量 500 万场景下，核心查询接口 P99 从 350ms 降至 22ms，
  系统 CPU 使用率降低 40%，大促零故障。」

## 0.3 AI / 算法工程师

### 核心技能词汇
**ML 通用**：特征工程 / XGBoost / LightGBM / 模型评估（AUC/NDCG/F1）/ A/B 实验
**深度学习**：PyTorch / TensorFlow / Transformer / 预训练模型 fine-tune
**NLP**：BERT/RoBERTa / 文本分类 / NER / 信息抽取 / 意图识别 / 机器翻译
**推荐系统**：双塔召回 / 精排（DeepFM/DCN）/ 特征存储 / 实时特征
**CV**：YOLOv8 / ResNet / 图像分类/分割/检测 / 数据增强
**LLM/大模型应用**：LangChain / LlamaIndex / RAG（向量检索+BM25混合）/ Prompt Engineering / Function Calling / Agent / RLHF 基础

### 量化指标参考
- 模型精度提升（AUC/F1/准确率，离线 vs 基线）
- 线上 A/B 指标（CTR/GMV/留存提升 %）
- 推理延迟（ms，P99）
- 训练数据规模（条数/GB）
- 服务规模（日调用次数/QPS）
- 内存/显存节省量

### 描述示例（LLM 应用方向）
- **弱**：「做了一个 RAG 系统。」
- **强**：「基于 LangChain + Weaviate 构建法律知识库问答系统，
  设计语义分段 + 关键词混合检索策略，
  召回准确率（Hit@5）较纯向量检索提升 31%，
  引入 Reranker 后回答质量人工评测提升 19 个点，
  服务月活用户 1200+，人工咨询量降低 45%。」

## 0.4 全栈开发

同时涵盖前端 + 后端技能，但需明确主攻方向。

建议写法：「以前端为主，具备 Node.js / Python 后端开发能力；
  有从零搭建前后端分离项目的完整经验。」

不建议写「精通前后端所有技术」，会让人觉得什么都会但什么都不深。

## 0.8 AI 应用工程师（LLM 应用层）

> 区别于 AI 算法工程师（侧重模型训练），AI 应用工程师侧重 LLM 的工程化落地。

### 核心技能词汇
**LLM 调用**：OpenAI API / Claude API（Anthropic SDK）/ 通义千问 / 文心一言 / 硅基流动 / LiteLLM
**提示工程**：Prompt Engineering / System Prompt 设计 / Few-shot / Chain of Thought / 结构化输出
**RAG 工程**：LangChain / LlamaIndex / 向量数据库（Chroma/Qdrant/Weaviate/Milvus/Pinecone）/ 文档解析 / Chunking 策略 / BM25 + 向量混合检索 / Reranker（交叉编码器）
**Agent 开发**：LangGraph / CrewAI / AutoGen / Dify / Coze / Function Calling / Tool Use / MCP Server 开发
**工程化**：流式输出（SSE）/ FastAPI / 上下文窗口管理 / Token 成本优化
**评测**：LangSmith / PromptFlow / RAGAS / 人工评测 + 自动评测

### 量化指标参考
- RAG 召回准确率（Hit@K / MRR）提升幅度
- 回答质量人工评测满意度（%）
- 人工客服咨询量降低比例
