# ATS 适配指南

本文档面向**投递外企 / 500 强在华分公司 / 留学生回国投外资企业**的求职者。
国内大多数公司（互联网/国企/中小企业）由 HR 人工筛简历，无需此指南。

---

## ATS 工作原理

ATS（Applicant Tracking System）在 HR 审阅之前自动处理简历：

```
求职者上传简历
      ↓
ATS 解析文件（PDF/Word）
      ↓
提取字段（姓名/联系方式/工作经历/教育/技能）
      ↓
提取关键词 → 与 JD 匹配打分
      ↓
过滤硬性条件（学历/年限/关键词缺失）
      ↓
HR 只看 ATS 初筛通过的简历
```

如果简历格式不被 ATS 识别，内容再好也可能被丢弃。

## 主流 ATS 系统及其特点

| ATS | 主要使用方 | 解析质量 | 特别注意 |
|-----|----------|---------|---------|
| Workday | 大型跨国公司（Google/IBM/德勤等）| 强 | 建议纯文字 PDF |
| Greenhouse | 科技公司/初创独角兽 | 强 | 支持多种格式 |
| Lever | 中型外企 | 较强 | - |
| Taleo（Oracle）| 传统大企业/金融/制造 | 中（老版本解析较弱）| 建议 .docx |
| iCIMS | 美国传统企业 | 中 | - |
| SAP SuccessFactors | 欧洲企业为主 | 中 | - |
| 北森（toB ATS 模式）| 部分中外资混合企业 | 中（中文识别较好）| - |
| Moka | 国内科技公司 / 外资在华 | 中 | - |

## 格式红线（会导致 ATS 解析失败）

### 绝对避免

1. **Word 表格布局**：用 Word 表格分栏做出的简历，ATS 会把表格内容读成一行乱码
2. **文本框**：Word 文本框（浮动框）里的内容 ATS 通常完全读不到
3. **双栏/多列布局**：ATS 从左到右、从上到下线性扫描，双栏会让两列内容混在一起
4. **扫描版 PDF**：没有文字图层，ATS 完全读不到任何文字
5. **图形化简历**：Canva、创客贴等做出的图片型简历无法被 ATS 解析

### 谨慎使用

6. **页眉 / 页脚中的联系信息**：部分 ATS 不读取页眉页脚区域
7. **图片 / 图标 / Logo**：图标本身读不到（如电话图标不等于"手机"），改用文字
8. **特殊字体 / 颜色方块**：部分 ATS 遇到无法渲染的格式会跳过该段
9. **非标准段落符号**：①②❶▪▶ 等特殊符号可能导致乱码，改用 • 或 -

## ATS 友好的格式原则

- **单栏布局**（从上到下线性阅读）
- **标准章节标题**（Work Experience / Education / Skills）
- **纯文字 PDF**（用 Word 或 Markdown 工具导出，而非扫描）
- **标准字体**（Calibri / Arial / Times New Roman / 微软雅黑）
- **不依赖颜色传达信息**（颜色装饰可以有，但关键内容不能只靠颜色区分）

## 关键词密度建议

### 为什么关键词密度重要

ATS 会将你简历中的词汇与 JD 匹配，关键词出现次数影响打分。

### 最优关键词布局

| 位置 | 优先级 | 建议 |
|-----|--------|------|
| 技能（Skills）章节 | ⭐⭐⭐ 最高 | ATS 优先扫描，关键词逐条列出 |
| 工作经历 bullet points | ⭐⭐⭐ 高 | 自然嵌入，每条经历体现 1-2 个关键词 |
| 简历摘要（Summary）| ⭐⭐ 中 | 集中出现 3-5 个核心关键词 |
| 教育背景 | ⭐ 低 | 专业名称本身就是关键词 |

### 关键词匹配原则

1. **原词嵌入**：ATS 通常不识别同义词（"Machine Learning" ≠ "ML"，两个都写）
2. **缩写和全称都写**："K8s / Kubernetes"、"CI/CD (Continuous Integration)"
3. **数量目标**：JD 中出现 3 次以上的词，简历里至少出现 2 次（技能 + 经历各 1 次）
4. **不要关键词堆砌**：有些求职者在简历末尾用白色字体堆砌关键词 —— 现代 ATS 有反作弊机制

## 文件命名规范

ATS 上传时，文件名本身可能被 HR 看到。推荐：

```
英文格式（外企推荐）：
FirstName-LastName-Resume.pdf
FirstName-LastName-JobTitle-Resume.pdf

中文格式（部分外资在华可用）：
姓名-岗位名称-简历.pdf
张明-数据工程师-简历.pdf
```

**避免**：
- ❌ `简历(2).pdf`（显示随意）
- ❌ `最终版最新20240301.pdf`（不专业）
- ❌ 中文文件名（部分系统上传后乱码）

---

## 中英双语简历注意点

如果需要准备中英双份简历：

1. **英文版另写**，不要直接翻译中文版（动词短语、量化方式、描述风格均不同）
2. 英文动词时态：在职用现在时（"Lead X team"），离职用过去时（"Led X team"）
3. 数字格式：用英文数字习惯（"$1.2M" 而非 "120 万美元"；"30%" 而非"百分之三十"）
4. 学历表述：用西方可识别格式
   - "Bachelor of Engineering in Computer Science, Peking University"
   - 而非"北京大学计算机科学与技术专业工学学士"
5. 公司名：中文公司英文名以官方为准（如"阿里巴巴集团" → "Alibaba Group"，不要自行翻译）

## ATS 通过后：人工审阅阶段

ATS 只是第一关。通过后，HR 会人工看简历。此时：
- 回归正常简历优化逻辑（参考 resume-editor / resume-experience 等）
- ATS 友好不代表 HR 友好（关键词密度不代表内容有说服力）
- 外企 HR 还会看：LinkedIn 主页 / GitHub / 作品集

> 核心：ATS 适配保证你不被机器过滤；人工审阅阶段才是真正说服的开始。
